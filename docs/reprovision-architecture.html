Zero-Touch Worker Reprovisioning

One click in the Hangar dashboard rebuilds a Firefox-CI Mac from scratch and returns it to service — no operator SSH, no manual steps. Two views: first where things live, then what happens, in order.

⚙️ Reprovision runner (macmini-m4-81)
Claims jobs from Hangar, then drives the full rebuild of the target machine. Runs up to 3 hosts at once (bandwidth-friendly).
→
🖥️ Target worker(s)
The Mac(s) being rebuilt. The runner drives each one on-host; the worker later fetches its own secrets (path below).
Why does the runner have to live inside the datacenter? Step 5 of the pipeline (SecureToken + Bootstrap Token) is an Apple security operation that, in this context, can only be performed over an on-host SSH session — so the driver must be on-network. That, more than anything, is why this is a datacenter runner and not just a Cloud Run job.
